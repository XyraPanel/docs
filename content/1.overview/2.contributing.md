XyraPanel is built by the community, for the community. We welcome anyone for fixing bugs, adding features, or improving the docs.

---

## Standards

- Follow existing code style
- Write tests for new features and fixes
- Update documentation when needed
- Use conventional commit messages
- Create a separate branch for each issue

## How to Contribute

1. Check existing issues before creating new ones
2. Report bugs or suggest features through issues
3. Fork and clone the repository
4. Create a branch for your changes
5. Make your changes and test thoroughly
6. Push to your fork and open a pull request
7. Respond to feedback during code review

---
The core team will review all pull requests and may suggest corrections or improvements to align with project standards. Feedback is part of the process and helps maintain code quality.
